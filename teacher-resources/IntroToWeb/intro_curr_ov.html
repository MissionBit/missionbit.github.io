MB Intro Curriculum Overview

Curriculum Learning Objectives: Web Dev

    • Make a web page from start to finish
      • Open an editor
      • Write or edit some HTML/CSS
      • Save the file(in the correct location)
      • Open/refresh the file in a browser
    • Manipulate basic styling (text/bg color, text weight, text size, hover underline)
    • Create two HTML documents that link to each other
    • Alter CSS in an external file, linked via tag
    • Download an image and embed it in an HTML document via tag
    • Add classes to HTML elements and use CSS to style them

      Description:


      Students will work in groups of no more than 3 to build a choose your own adventure game, where the choices are represented in a series of interlinked HTML documents.


      Skeleton/Structure:

      • 4 pages total
      • One page page linking to three distinct choices
      • Each page should link to a shared external CSS file
      • Each page should include an embedded (dummy) image

      Instruction to Students:

      • Students should work in groups of no less than 2 and no more than 3
      • Each student is responsible for one "branch" of the story
      • Each student needs to create at least 6 choices, linked to in whatever order
      • Groups should take 15-20 minutes to draw out the choice tree and divvy up the work.
      • Each group will present their game to the class discussing:
        • Who did what
        • What html/css elements did you use?
        • challenges faced
        • Provide a demo of their adventure game

      Instructor Duties

      • Review each group's draft of their game before approving computer use.
      • Ensure that students are following project specifications
      • Provide feedback on group presentations
  • Given a pre-existing piece of code, students will be able to identify the data type of numerical, string, and boolean variables.
  • Students will be able to create a single-branch conditional statement
  • Students will be able to write a simple functions that accepts a single argument
  • Students will be able to loop over an array and print out the values in the array
  • Students will be able to loop over an array and print out a modified version of the values in the array (e.g., loop over an array of numbers and print out each number times two)
  • Students will be able to explain why we use JS in web pages
  • Understand basic game concepts
    • Game States
    • Sprites
    • players/avatars
    • mechanics
    • dynamics
    • goals
    • theme
  • Be able to understand the flow of game development
  • Build a functional game to be presented at Demo Day
  • Increased confidence in public speaking, communication, and collaboration
